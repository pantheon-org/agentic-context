# ANALYSIS: Understand-Anything

---

## Summary

Understand-Anything is a TypeScript/Node.js Claude Code skill (slash commands `/understand` and `/understand-domain`) that orchestrates a 5-agent pipeline to produce an interactive in-IDE knowledge graph dashboard from a codebase. Its core differentiator is a dual-graph view: a structural graph (files, functions, classes) and a domain graph (business flows, processes, steps) generated by a 6th `domain-analyzer` agent. The tool targets developer comprehension and onboarding rather than agent token reduction — no token metrics are claimed. With 8,081 GitHub stars and MIT license it has significant community traction, but key operational details (LLM API call costs during indexing, supported language list, dashboard persistence format) are undocumented.

---

## What it does (from reference documentation)

### Multi-agent pipeline

Six agents invoked via slash command:

| Agent | Role |
|-------|------|
| `project-scanner` | Discover files, detect languages and frameworks |
| `file-analyzer` | Extract functions, classes, imports; produce graph nodes and edges |
| `architecture-analyzer` | Identify architectural layers |
| `tour-builder` | Generate guided learning tours ordered by dependency |
| `graph-reviewer` | Validate graph completeness and referential integrity |
| `domain-analyzer` | Extract business domains, flows, and process steps (via `/understand-domain`) |

File analyzers run in parallel (up to 5 concurrent, 20–30 files/batch). The `graph-reviewer` runs inline by default; `--review` flag triggers full LLM-backed validation.

### Interface

- Slash commands: `/understand` (structural + architectural analysis), `/understand-domain` (business domain extraction)
- Multi-platform MCP installation: Claude Code (`claude mcp add`), Codex, Cursor, VS Code + Copilot, Gemini CLI, OpenCode, Antigravity
- Output: interactive in-IDE dashboard with structural graph, domain graph, fuzzy+semantic search, diff impact analysis, and guided tours

### Incremental updates

Only changed files since the last run are re-analyzed. No latency or throughput benchmarks are provided in the README.

## Architectural assessment

### What's genuinely novel

1. **Domain graph view.** The `domain-analyzer` agent extracts a business domain ontology — flows, processes, steps — and maps code to them. No other tool in this survey offers a code-to-business-process graph. This is the most differentiated capability and the primary reason to consider this tool over structural-only alternatives.

2. **Persona-adaptive UI.** The dashboard adjusts detail level based on stated persona (junior developer, PM, power user). This is an unusual product decision for a developer tool and reflects the human-comprehension focus.

3. **Dependency-ordered guided tours.** The `tour-builder` agent generates learning tours that sequence architectural concepts in dependency order — a structured onboarding path rather than a raw graph dump.

4. **Multi-platform MCP deployment.** First-class installation paths for 7+ platforms including non-Claude runtimes (Gemini CLI, OpenCode). The widest platform coverage of any tool surveyed.

### Gaps and risks

- **LLM API calls during indexing are likely but undisclosed.** The file analyzer extracts functions, classes, and imports via agents — this almost certainly requires live LLM calls for each file batch, implying per-run API cost and latency proportional to codebase size. The README does not state this or provide cost estimates.
- **Supported language list is unspecified.** The tool claims to handle "any codebase" but relies on LLM-based file analysis rather than deterministic AST parsing. Accuracy for poorly named or undocumented codebases is unknown.
- **Dashboard persistence format is undocumented.** It is unclear whether the generated graph survives IDE restarts or requires re-running `/understand` each session.
- **Domain graph quality is LLM-dependent.** Business domain extraction from code is a hard inference problem. Quality will degrade for codebases with poor naming conventions or absent documentation, and no accuracy benchmark exists.
- **`graph-reviewer` is opt-in for full validation.** The default inline reviewer may miss graph errors; `--review` adds cost and latency that are not quantified.
- **No benchmark against comprehension outcomes.** The tool's goal is developer understanding, but there is no study or metric measuring whether tours, domain graphs, or persona-adaptive UI actually reduce onboarding time or error rates.

---

## Recommendation

**Evaluate for onboarding and domain mapping use cases.** If the primary need is helping a developer (or agent) understand what a codebase *does* in business terms — not just how it is structured — the `domain-analyzer` / `/understand-domain` path offers capability unavailable elsewhere in this survey.

**Do not use as a substitute for structural graph queries.** For call-path tracing, impact analysis, and token-efficient structural lookup, codebase-memory-mcp's deterministic AST graph is more reliable and has lower per-query cost.

**Clarify LLM API cost before adopting.** The likely per-run API cost for large codebases is the most important unknown. Run `/understand` on a mid-sized project with API usage logging before committing to this tool in cost-sensitive agent workflows.

**Treat domain graph output as a starting point.** Business process extraction from code is inherently heuristic. Use the domain graph to orient, then verify key process claims against actual source.
